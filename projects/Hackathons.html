MakeCU Hackathon

Columbia University · 2025–2026

Team: CUPu
Project: Computer-vision-based robotic trash and recycling sorter

Inspiration

Environmental sustainability is a growing global challenge. Our team wanted to create a system that makes sustainable behavior easier and more accessible, leading to the concept of an automated garbage sorting robot.

What We Built

A robotic system that detects when an object is placed in front of it, classifies the object as trash or recyclable using computer vision, and mechanically directs it into the correct bin.

Final Product Demo

How We Built It

Hardware

  • Raspberry Pi as the main controller
  • Camera and ultrasonic sensor for object detection and positioning
  • Servo motor and driver for mechanical sorting
  • 3D-printed platform and electronics mount designed in SolidWorks
  • Carbon-based bin and funnel structure

Software

  • Computer vision model trained on a labeled Kaggle dataset
  • Python-based image processing and inference pipeline
  • Real-time classification of recyclable vs. trash items

Challenges

  • Unreliable Raspberry Pi networking over mobile hotspot
  • Long model training times with limited bandwidth
  • Hardware failure requiring rapid redesign and reprogramming
  • Limited access to batteries and 3D printers during the event

Accomplishments

  • Trained a computer vision model achieving ~96% accuracy
  • Successfully integrated hardware and software into a working prototype
  • Delivered a functional system under extreme time and resource constraints

Skills & Tools

  • Programming: Python, NumPy, OpenCV, PyTorch
  • Hardware: Raspberry Pi, GPIO, Sensors, Servos
  • Design: SolidWorks, Rapid Prototyping
  • Skills: Teamwork, Debugging, Time Management

HemiSec: Portable Biometric Vein Scanner

Hackathon Project · Team of 3 · Hardware Lead

Team & Role

HemiSec was developed as a three-person interdisciplinary team focused on creating a secure and portable biometric authentication system. I primarily contributed to the hardware development, including system architecture, component integration, enclosure design, and physical prototyping.

Our goal was to create a convenient authentication method that uses a person's unique palm vein pattern — a biometric identifier that is significantly more difficult to replicate or spoof compared to traditional passwords, cards, or fingerprints.

Hardware System

The portable scanner uses near-infrared imaging to capture the unique vein patterns beneath the user's palm. The hardware system was designed around a Raspberry Pi platform capable of both image capture and onboard processing.

  • Processing: Raspberry Pi 3 for image capture and computation
  • Camera: Arducam OV5647 camera module
  • Imaging: 850nm infrared illumination panel to reveal palm veins
  • Enclosure: Lightweight, durable PLA case designed for portability
  • Networking: Router-based connection for communication between devices

System Workflow

1. Enrollment

A new user creates an account through the application interface. The palm vein scanner captures an infrared image of the user's palm and sends the image through the processing pipeline to generate a unique biometric representation.

2. Image Processing Pipeline

Raw infrared images are enhanced using multiple image processing filters to improve vein visibility. The processed image is converted into a vein feature map, allowing important biometric features within the region of interest (ROI) to be extracted.

3. Authentication

During authentication, the Raspberry Pi captures a new palm scan and repeats the same enhancement and feature extraction pipeline. Extracted features are compared against encrypted biometric data stored on the backend to determine identity.

4. User Interface

A simple mobile-style interface allows users to quickly enroll, authenticate, and manage their biometric profiles. The design prioritizes accessibility, speed, and ease of use.

Design Process & Engineering Decisions

The design process focused on balancing security, portability, and reliability. The Raspberry Pi platform was selected because it provided sufficient processing capability while maintaining a compact form factor. The PLA enclosure was designed to protect the electronics while keeping the scanner lightweight and portable.

Future additions: explain CAD iterations, enclosure design decisions, mounting methods, hardware testing, and prototype improvements.

Applications & Impact

A portable palm vein authentication system provides a secure and convenient alternative to traditional authentication methods.

  • User Convenience: Contactless authentication eliminates the need to remember passwords or carry physical authentication tokens.
  • Portability: A compact design enables deployment in environments such as access control, mobile devices, and secure verification systems.
  • Security: Palm vein patterns are internal biological features, making them significantly harder to replicate compared to surface-level biometrics.

Limitations

  • Vein detection may become less reliable for individuals with worn or calloused skin.
  • Environmental lighting variations may affect image quality and accuracy.
  • The IR illumination system requires an external power source.
  • Infrared imaging can introduce image distortion requiring additional processing.

Future Development

  • Design custom PCB components to further reduce system size.
  • Create a modular enclosure compatible with different phone platforms.
  • Improve camera selection and image capture quality.
  • Develop a more integrated external power solution.
  • Improve integration with Apple iOS devices.
